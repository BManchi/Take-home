# Fidelity

**Source:** https://help.copilot.money/en/articles/9828928-fidelity

Fidelity accounts are connected via Akoya. As a result, the behavior for these accounts is slightly different than accounts connected through other data aggregators.

---

# Account Updates

The Akoya / Fidelity integration surfaces new data once a day, with a one day delay. This means it is expected that there is a one day delay between what you see in Copilot compared to what you see on the Fidelity website or app.

# Pending Transactions

At this time we only receive transaction data once it has posted to your account, not pending transaction data. We have requested Fidelity provide pending transaction data, but we cannot provide an ETA at this time.

# Supported Account Types

Akoya only supports surfacing investment, credit card, depository, and loan accounts. **Crypto** and **pension** accounts aren't expected to surface right now via Akoya because Fidelity is not sharing them, but we've requested Fidelity consider sharing these based on user feedback as well.

# Switching Account Types

In Copilot, transactions do not surface for Investment accounts. For Fidelity accounts specifically, our support team can change the typing of these accounts from Investment to Depository. This allows us to surface transactions for these accounts, which is useful if you use an investment account primarily as a checking account. Please note, switching an account to the depository type will hide any investment holdings for that account as well.

If you'd like us to switch an investment account to depository to surface transaction data, please reach out to us via the in-app chat and a member of our Customer Success team would be happy to help.
